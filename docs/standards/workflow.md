# 🔄 工作流程规范

本规范定义足球比赛预测分析的完整工作流程，确保每个环节的标准化和可追溯性。

---

         ↓

## Step 1: 赛前准备

### 时间节点
- **比赛前7天**: 确定目标比赛列表
- **比赛前3天**: 收集球队基本信息
- **比赛前1天**: 完成数据采集准备

### 任务清单
- [ ] 从赛程表中筛选重点比赛
- [ ] 确认比赛时间和对阵双方
- [ ] 检查球队近期状态
- [ ] 初步评估比赛重要性

---

## Step 2: 数据采集

### 使用Agent
数据采集Agent (`data_collector_agent.md`)

### 时间节点
- **比赛前24小时**: 采集初始数据
- **比赛前12小时**: 采集中期数据
- **比赛前2小时**: 采集临盘数据（重要！）

### 数据来源优先级
1. **竞彩网** - 官方赔率
2. **澳客官网** - 凯利指数
3. **500彩票网** - 欧赔对比
4. **搜索引擎** - 补充信息

### 任务清单
- [ ] 访问竞彩网获取胜平负赔率
- [ ] 访问澳客获取凯利指数
- [ ] 访问500彩票网对比验证
- [ ] 搜索首发阵容和伤病名单
- [ ] 记录多家公司数据
- [ ] 数据验证（多源对比）
- [ ] 保存结构化数据

### 输出文件
- `data/odds/YYYYMMDD-主队-客队.json`
- `data/teams/主队-injuries.md`
- `data/teams/客队-injuries.md`

---

## Step 3: 比赛分析

### 使用Agent
比赛分析Agent (`match_analyzer_agent.md`)

### 分析维度
- [ ] 球队基本信息（排名、积分）
- [ ] 近期状态（近10场、近5场）
- [ ] 攻防能力（场均进球、失球）
- [ ] 战术打法（阵型、风格）
- [ ] 历史交锋（近6次交手）
- [ ] 伤病情况（伤停球员）
- [ ] 球队战意（联赛形势）
- [ ] SWOT分析

### 输出文件
- `analysis/match/YYYYMMDD-主队-客队.md`

---

## Step 4: 赔率分析

### 使用Agent
赔率分析Agent (`odds_analyzer_agent.md`)

### 分析内容
- [ ] 凯利指数分析（一致性、离散度）
- [ ] 赔率变化趋势（初盘→临盘）
- [ ] 亚盘盘口变动（升盘/降盘）
- [ ] 大小球泊松分布计算
- [ ] 庄家心理识别（资金流向）
- [ ] 凯利口诀验证

### 输出文件
- `analysis/odds/YYYYMMDD-主队-客队.md`

---

## Step 5: 综合预测

### 预测要素权重

| 要素 | 权重 | 说明 |
|------|------|------|
| 基本面分析 | 35% | 球队实力、状态、攻防 |
| 赔率分析 | 30% | 凯利指数、盘口变动 |
| 历史交锋 | 15% | 交锋记录、心理优势 |
| 战意分析 | 10% | 联赛形势、比赛重要性 |
| 伤病情况 | 10% | 核心球员伤停 |

### 预测报告内容
1. 比赛基本信息
2. 基本面分析摘要
3. 赔率分析摘要
4. 综合评分对比
5. 胜平负概率
6. 比分预测（Top 3）
7. 大小球预测
8. 让球盘预测
9. 投注建议
10. 风险提示

### 输出文件
- `reports/YYYYMMDD-主队-客队.md`

---

## Step 6: 记录预测

### 时间节点
- **比赛开始前**: 完成预测记录

### 记录内容
- [ ] 填写预测数据表
- [ ] 分配唯一预测ID
- [ ] 记录所有预测字段
- [ ] 关联分析报告链接
- [ ] 数据格式验证

### 输出文件
- `europe_leagues/[联赛]/analysis/predictions/predictions_YYYY-MM.md`

---

## Step 7: 比赛进行

### 关注内容
- [ ] 首发阵容确认
- [ ] 比赛进程实时关注
- [ ] 关键事件记录（进球、红牌）
- [ ] 战术变化观察

---

## Step 8: 结果记录

### 使用Agent
结果追踪Agent (`result_tracker_agent.md`)

### 时间节点
- **比赛结束后24小时内**: 完成结果记录

### 数据来源
1. **竞彩网** - 官方结果
2. **ESPN** - 国际比赛结果
3. **各大体育媒体** - 交叉验证

### 任务清单
- [ ] 收集实际比赛结果
- [ ] 多源验证结果准确性
- [ ] 填写结果数据表
- [ ] 关联预测ID
- [ ] 标记预测是否正确
- [ ] 记录比分、进球数
- [ ] 数据归档

### 输出文件
- `europe_leagues/[联赛]/analysis/results/results_YYYY-MM.md`

---

## Step 9: 月度统计

### 时间节点
- **每月最后一天**: 执行月度统计

### 统计内容
- [ ] 运行 `analyze_prediction_accuracy.py`
- [ ] 计算总体准确率
- [ ] 分联赛准确率对比
- [ ] 预测类型准确率
- [ ] 时间趋势分析
- [ ] 偏差分析
- [ ] 生成准确率报告

### 输出文件
- `reports/monthly/YYYY-MM-accuracy-report.md`

---

## Step 10: 持续优化

### 优化内容
- [ ] 分析预测偏差原因
- [ ] 识别系统性问题
- [ ] 调整分析权重
- [ ] 优化预测模型
- [ ] 更新Agent文档
- [ ] 测试改进效果

### 优化周期
- **月度**: 小调整
- **季度**: 大优化
- **年度**: 全面复盘

---

## 质量检查清单

### 赛前阶段
- [ ] 数据采集完整（所有数据源）
- [ ] 数据验证通过（多源一致）
- [ ] 基本面分析覆盖所有维度
- [ ] 赔率分析包含凯利指数
- [ ] 预测报告格式规范

### 赛后阶段
- [ ] 结果记录及时（24小时内）
- [ ] 结果验证通过（多源一致）
- [ ] 预测与结果关联正确
- [ ] 准确率统计完整
- [ ] 优化建议合理

---

## 异常处理

### 数据异常
- **问题**: 数据源访问失败
- **处理**: 使用备用数据源，延迟记录

### 比赛延期
- **问题**: 比赛延期或取消
- **处理**: 更新预测记录状态，待重新安排后分析

### 预测偏差大
- **问题**: 连续预测错误
- **处理**: 立即进行偏差分析，调整模型

---

## 最佳实践

1. **提前准备**: 比赛前3天开始准备
2. **临盘重要**: 比赛前2小时必须再次采集数据
3. **多源验证**: 至少3个数据源交叉验证
4. **及时记录**: 预测和结果都要及时记录
5. **定期复盘**: 每月进行准确率分析和优化
6. **文档完善**: 所有分析都要有文档记录
