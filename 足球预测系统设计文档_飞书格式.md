## 五、端到端正式流程

### 5.1 赛前流程

1. `collect-data` 获取赛程、`match_id` 与上下文
2. `predict-match` / `predict-schedule` 执行预测
3. `domain/persistence.py` 统一处理 SoT / runtime-only 写回、archive、MEMORY、RAG、registry 等 side effects
4. 需要阶段化审计时，使用 `harness-run`

### 5.2 赛后流程

1. `pending-results` 查看待回填比赛
2. `auto-sync-results` 尝试自动同步
3. 自动未命中时使用 `save-result` 手工补录
4. 需要批次复盘时走 `sync-pending-results-review`
5. 需要显式重建时再执行 `accuracy --refresh`

### 5.3 自动闭环事实

正常赛果命中后，系统会继续推动多项衍生产物刷新，包括：

- SoT 比分与备注
- `MEMORY.md`
- `prediction_archive.json`
- 准确率统计
- RAG 样本与索引
- review-learning 相关输出

因此 `accuracy --refresh` 是显式重建入口，而不是唯一常规路径。

## 七、示例命令

### 7.1 基础健康检查

```bash
cd /Users/bytedance/trae_projects/europe_leagues
python3 prediction_system.py list-leagues --json
python3 prediction_system.py health-check --json
```

### 7.2 采集赛程数据

```bash
python3 prediction_system.py collect-data --league la_liga --date 2026-05-14 --json
```

### 7.3 单场比赛预测

```bash
python3 prediction_system.py predict-match \
  --league la_liga \
  --home-team 塞尔塔 \
  --away-team 莱万特 \
  --date 2026-05-13 \
  --match-id 1324404 \
  --json
```

### 7.4 Harness 编排

```bash
python3 prediction_system.py harness-run \
  --pipeline match_prediction \
  --league la_liga \
  --home-team 塞尔塔 \
  --away-team 莱万特 \
  --date 2026-05-13 \
  --json
```

### 7.5 赛果回填

```bash
python3 prediction_system.py pending-results --days-back 30 --json
python3 prediction_system.py auto-sync-results --json
python3 prediction_system.py save-result --match-id la_liga_20260513_塞尔塔_莱万特 --home-score 2 --away-score 1 --json
python3 prediction_system.py sync-pending-results-review --days-back 30 --limit 20 --json
python3 prediction_system.py accuracy --refresh --json
```
