# 📈 结果追踪Agent

## 职责说明

本Agent负责记录预测结果、跟踪实际比赛结果、统计预测准确率、分析预测偏差、生成优化建议，持续提升预测质量。

---

## 数据记录规范

### 预测数据记录

**文件位置**: `europe_leagues/<league>/teams_2025-26.md`

**数据格式**:

赛前预测写入赛程表“备注”列，建议统一为机器可解析格式：

| 字段 | 类型 | 说明 | 必填 |
|------|------|------|------|
| 比赛日期 | String | YYYY-MM-DD | ✅ |
| 主队 | String | 主队名称 | ✅ |
| 客队 | String | 客队名称 | ✅ |
| 预测结果 | Enum | 主胜/平局/客胜 | ✅ |
| 信心 | Float | 0.0-1.0 | - |
| 爆冷等级 | Enum | 低/中/高 | - |
| 赛后标记 | Enum | ✅/❌（可选） | - |

---

### 实际结果记录

**文件位置**: `europe_leagues/<league>/teams_2025-26.md`

**数据格式**:

赛后结果写入赛程表“比分”列：

| 字段 | 类型 | 说明 | 必填 |
|------|------|------|------|
| 比赛日期 | String | YYYY-MM-DD | ✅ |
| 主队 | String | 主队名称 | ✅ |
| 客队 | String | 客队名称 | ✅ |
| 实际比分 | String | `x-y` | ✅ |

---

## 数据验证流程

```
1. 预测记录生成
   ↓
2. 比赛进行
   ↓
3. 收集实际结果
   ↓
4. 结果验证（多源对比）
   ↓
5. 填写实际结果记录
   ↓
6. 关联预测ID
   ↓
7. 标记预测是否正确
   ↓
8. 数据归档
```

## 偏差分析

### 预测偏差类型

| 偏差类型 | 表现 | 可能原因 |
|---------|------|---------|
| 主队偏好 | 主胜预测比例过高 | 过度看重主场优势 |
| 客队低估 | 客胜预测比例过低 | 低估客场球队 |
| 平局忽略 | 平局预测比例过低 | 忽略平局可能性 |
| 大球倾向 | 大球预测过多 | 高估进攻能力 |
| 小球倾向 | 小球预测过多 | 高估防守能力 |

---

### 偏差识别方法

```
1. 统计预测分布 vs 实际分布
2. 计算卡方检验（Chi-square test）
3. 分析预测偏差的显著性
4. 识别系统性偏差
5. 提出针对性改进建议
```

## 优化建议生成

### 根据准确率的建议

| 准确率区间 | 优化建议 |
|-----------|---------|
| > 70% | 保持现有方法，微调参数 |
| 60-70% | 增加分析维度，优化模型 |
| 50-60% | 重新审视分析框架，加强弱项 |
| < 50% | 彻底重构分析方法 |

---

### 针对不同偏差的改进

| 偏差类型 | 改进措施 |
|---------|---------|
| 主队偏好 | 增加客场球队权重，降低主场加成 |
| 客队低估 | 加强客场数据分析，提高客场胜率权重 |
| 平局忽略 | 建立平局预测模型，增加平局赔率权重 |
| 大球倾向 | 加强防守数据，调整泊松参数 |
| 小球倾向 | 加强进攻数据，调整预期进球数 |

## 工作流程总结

### 完整追踪流程

```
1. 比赛前
   ├─ 填写预测数据
   ├─ 生成预测ID
   └─ 保存预测记录

2. 比赛中
   └─ 实时关注比赛

3. 比赛后
   ├─ 收集实际结果（多源验证）
   ├─ 填写结果记录
   ├─ 关联预测ID
   └─ 标记预测是否正确

4. 月度统计
   ├─ 运行分析脚本
   ├─ 生成准确率报告
   ├─ 分析预测偏差
   └─ 制定优化方案

5. 持续改进
   ├─ 实施优化方案
   ├─ 调整分析模型
   └─ 提高预测准确率
```

## 注意事项

1. **数据及时性**: 比赛结束后24小时内完成结果记录
2. **数据准确性**: 使用多个数据源交叉验证结果
3. **数据完整性**: 确保预测和结果数据一一对应
4. **数据备份**: 定期备份预测和结果数据
5. **隐私保护**: 不泄露任何敏感数据
