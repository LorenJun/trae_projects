# 当前预测流程完整解析

本文是仓库根层面的流程说明，描述 `europe_leagues/` 当前正式预测主链。

## 入口与权威链

对 `europe_leagues/` 子项目，入口分两层：

- `europe_leagues/prediction_system.py`：兼容 / 发现入口
- `europe_leagues/app/cli.py`：真实 CLI 路由、参数定义与 JSON 输出入口

不要把 `prediction_system.py` 误判成业务主逻辑实现。

## 当前正式主流程

```text
1. 确认比赛信息
2. collect-data / 赛程抓取定位 match_id
3. predict-match / predict-schedule
4. 检查真实盘口、RAG 与关键诊断字段
5. 按比赛类型写回 SoT 或 runtime-only 归档
6. save-result / auto-sync-results / result-sync-daemon / sync-pending-results-review 回填赛果
7. 结果闭环自动刷新 archive / MEMORY / RAG / review-learning / accuracy 等衍生产物
8. 需要显式重建时，再执行 accuracy --refresh
```

## SoT 与 runtime-only 边界

### SoT-backed competitions

以下 competition 以 markdown SoT 为主：

- `premier_league`
- `la_liga`
- `serie_a`
- `bundesliga`
- `ligue_1`
- `world_cup`

写回位置：

- 五大联赛：`europe_leagues/<league>/teams_2025-26.md`
- 世界杯：`europe_leagues/world_cup/teams_2026.md`

### runtime-only competitions

以下 competition 主要依赖 runtime archive 与滚动记忆：

- `europa_league`
- `champions_league`
- `conference_league`
- 其他杯赛 / 欧战扩展比赛

写回位置：

- 项目根 `MEMORY.md`
- `europe_leagues/.okooo-scraper/runtime/*.json`

## 关键 owner

- CLI 命令 owner：`europe_leagues/app/cli.py`
- 预测持久化 owner：`europe_leagues/domain/persistence.py`
- 结果同步 owner：`europe_leagues/runtime/result_sync.py`
- 结果归档 / 准确率 owner：`europe_leagues/result_manager.py`
